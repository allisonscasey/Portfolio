Overview

This project focused on building a working technical system under real constraints. The challenge was not just making the prototype function once, but understanding what conditions it needed to handle and how to validate that it worked reliably.

This page can become the full story behind the build: where the requirements came from, what assumptions shaped the design, and how the final version improved through testing.

What I Did

Process

I started by separating the project into what had to be true for the system to work: inputs, outputs, constraints, failure modes, and success criteria. That made the build easier to reason about before getting too deep into implementation.

Each iteration answered a specific question. Could the prototype meet the basic requirement? Could it do so consistently? What changed when the environment, inputs, or assumptions shifted?
